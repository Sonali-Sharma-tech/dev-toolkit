## Content Guidelines

### Style
- Copy-paste ready commands (minimal placeholders)
- Include "when to use" context for each command
- Real-world examples, not theoretical
- Note macOS vs Linux differences where applicable
- No fluff, concise explanations

### Structure
Each documentation file should have:
1. Clear title and description
2. Table of Contents
3. Organized sections with `---` separators
4. Code blocks with language specified (```bash, ```json, etc.)
5. Brief explanation after each command
